Tabla virtualizada beta #

Junto con la evolución del desarrollo web, el componente de tabla siempre ha sido el componente más popular en nuestras aplicaciones web, especialmente para paneles, análisis de datos. Para tabla V1, con incluso 1000 registros de datos, puede ser muy molesto al usarlo, por el pobre rendimiento.

Con la Tabla Virtualizada, puede renderizar masivos trozos de datos en un parpadeo de ojos.

TIP

Este componente todavía está bajo prueba, úselo bajo su propio riesgo. si encuentra algún error o problema, por favor infórmenos en GitHub para que lo arreglemos. También hay algunas APIs que no se mencionan en esta documentación, Algunas de ellas todavía no están completamente desarrolladas, por lo que no las mencionamos aquí.

Aunque la tabla virtualizada es eficiente, cuando la carga de datos es demasiado grande, su red, tamaño de memoria, etc. pueden ser el cuello de la botella. Así que tenga en cuenta que la Tabla Virtualizada nunca es la solución definitiva para todo, considere paginar sus datos, añadir filtros, etc.

Uso básico #

Vamos a renderizar un caso básico de Tabla Virtualizada con 10 columnas por 1000 filas, para demostrar su rendimiento.

Auto redimensionar #

Cuando vea que no puede pasar la propiedad width y height a la tabla, puede utilizar el componente AutoResizer para envolver el componente de tabla y actualizará el ancho y la altura automáticamente.

Redimensione su navegador para ver cómo funciona.

TIP

Asegúrese de que el nodo padre de AutoResizer tiene un HEIGHT FIJO, porque el valor de altura se estableció a 100% por defecto. También puede configurarlo pasándolo a través del atributo style a AutoResizer.

Personalizar renderizador de celdas #

Por supuesto, usted puede renderizar la célula de tabla según sus necesidades, aquí hay un ejemplo simple de cómo personalizar la celda.

Tabla con selecciones #

Utilizando el renderizador de celdas personalizado para permitir la selección para la tabla.

Edición en línea #

Al igual que las selecciones que hemos demostrado anteriormente, puede utilizar el mismo método para activar la edición en línea.

Tabla con estado #

Puede destacar su contenido de tabla para distinguir entre "success, information, warning, danger" y otros estados.

Use row-class-name para personalizar cómo se ve la fila. En este caso, cada fila 10 será resaltada con la clase bg-blue-200, cada quinta fila será resaltada con la clase bg-red-100.

Tabla con filas pegajosas #

Puede hacer que algunas filas se peguen en la parte superior de la tabla, y eso se puede lograr muy fácilmente usando el atributo fixed-data.

Puede agregar dinámicamente el registro con los eventos de desplazamiento como se hace en este ejemplo.

Tabla con columnas fijas #

Si por alguna razón, quiere hacer que las columnas se peguen a la izquierda y la derecha, puede hacerlo añadiendo atributos especiales para la tabla.

Puede establecer el atributo de la columna fixed a true (representando FixedDir.LEFT) o FixedDir.LEFT o FixedDir.RIGHT

Encabezado de grupo #

Al personalizar la renderización de cabecera puede agrupar su cabecera como se hace en este ejemplo.

TIP

En este caso hemos usado JSX que no está soportado en el playground, puede probarlo en su entorno local o en un IDE online como codesandbox.

Se recomienda que escriba el componente de tabla en JSX, ya que contiene manipulaciones de VNode.

Filtros #

Las tablas virtualizadas proveen de renderizaciones personalizadas de las cabeceras y pueden ser usados para renderizar filtros

Ordenable #

Puede ordenar la tabla con el estado de ordenación.

Orden controlado #

Puede definir múltiples columnas ordenables cuando lo necesite. Tenga en cuenta que si define múltiples columnas ordenables, la interfaz de usuario puede parecer extraña para sus usuarios, ya que no está claro qué columna está siendo ordenada.

Intervalo cruzado #

Cuando la lista es grande, y a veces se pierde en qué fila y columna se está visitando, usar esto es extremadamente útil.

Datos con formato de árbol #

La tabla virtual también puede representar datos como árbol, puede expandir/contraer el nodo del árbol haciendo clic en el icono de flecha.

Filas de altura dinámica #

La Tabla Virtual también puede renderizar filas con altura dinámica, cuando tiene datos sin saber qué tamaño tendrá el contenido, puede querer usar esta función para renderizar una fila de altura dinámica. Debe pasar estimated-row-height para habilitar esta función, y cuanto más cerca esté de la altura estimada, más suave será el renderizado.

TIP

La altura de cada fila se mide dinámicamente al procesar las filas, por lo que la interfaz de usuario puede estar rebotando si está intentando renderizar una gran cantidad de datos.

Vista de detalle #

El renderizado de altura dinámica podemos usarlo para renderizar detalles en la tabla.

Pie personalizado #

Renderice un pie de página personalizado cuando quiera mostrar un mensaje o información concluyente.

Overlay #

Procese una superposición sobre la tabla cuando quiera mostrar un indicador de carga o cualquier otra cosa.

Desplazamiento manual #

Utilice los métodos expuestos por la Tabla V2 para desplazarse manualmente/programáticamente con las desplazamientos/filas deseadas.

TIP

El segundo parámetro para scrollToRow es la estrategia de desplazamiento que por defecto es auto, calcula la posición para desplazarse por sí misma. Puede pasar su propia estrategia si quiere desplazarte a una posición específica. Las opciones disponibles son "auto" | "center" | "end" | "start" | "smart"

La diferencia entre smart y auto es que auto es un subconjunto de la estrategia de desplazamiento smart.

FAQs #

¿Cómo renderizo una lista con la casilla de verificación en la primera columna? #

Ya que se le permite definir su propio renderizado de celdas, puede hacer lo que se hace en el ejemplo Renderizado Personalizado de Celdas para renderizar la casilla de verificación por usted mismo, y mantener el estado usted mismo.

Por qué la tabla virtualizada proporciona menos características que TableV1 #

Para la tabla virtualizada, pretendemos proporcionar menos características y dejar que nuestros usuarios implementen sus propias características por demanda. Integrar demasiadas características hace que el código sea difícil de mantener y para la mayoría de los usuarios las características básicas son suficientes. Algunas características clave aún no han sido desarrolladas. Nos encantaría saber de ti. Únete a Discord para estar atento.
