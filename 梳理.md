## 一、项目结构总览

这是一个基于 **pnpm workspace** 的 monorepo，核心结构如下：

```
gubo-ui/
├── packages/                  # 核心源码包
│   ├── element-plus/          # ★ 主包（最终发布的包）
│   ├── components/            # 80+ Vue 组件（每个组件一个文件夹）
│   ├── theme-chalk/           # 主题样式（SCSS 源码）
│   ├── hooks/                 # Vue composables
│   ├── directives/            # Vue 指令（v-loading 等）
│   ├── locale/                # 国际化语言包
│   ├── constants/             # 共享常量
│   ├── utils/                 # 工具函数
│   └── test-utils/            # 测试辅助
├── internal/                  # 内部构建工具
│   ├── build/                 # ★ 核心构建流程（Rolldown）
│   ├── build-utils/           # 路径、命令等构建辅助
│   ├── build-constants/       # ★ 包名常量（PKG_NAME 等）
│   ├── metadata/              # 元数据生成
│   └── eslint-config/         # ESLint 配置
├── scripts/                   # 构建/发布脚本
├── docs/                      # VitePress 文档站
├── play/                      # 开发调试用的 Vite 项目
├── typings/                   # 全局类型声明
└── dist/                      # ★ 构建输出目录
```

### 关键子包关系

`packages/element-plus` 是**最终发布的唯一包**，它聚合了所有子包：

```
element-plus/index.ts
  └─ re-export ─→ @element-plus/components（80+ 组件）
  └─ re-export ─→ @element-plus/constants
  └─ re-export ─→ @element-plus/directives
  └─ re-export ─→ @element-plus/hooks
  └─ make-installer（Vue app.use 安装器）
```

子包（`@element-plus/components` 等）**不会单独发布到 npm**，它们在构建时被打包进主包的 `es/` 和 `lib/` 目录中。

## 二、构建流程详解

运行 `pnpm build` 实际执行的是 `pnpm run -C internal/build start`，即 `jiti internal/build/buildfile.ts`。

### 构建步骤

```
1. makeOutput()         → 清空 dist/，创建 dist/element-plus/
2. buildHelper()        → 生成 web-types.json（IDE 支持）
3. 并行执行:
   ├─ generateTypesDefinitions → TypeScript 类型声明 (.d.ts)
   ├─ buildModules             → ESM (.mjs) + CJS (.js) 按组件拆分
   ├─ buildFullBundle          → UMD/ESM 完整打包（CDN 用）
   ├─ buildStyle               → SCSS → CSS（theme-chalk）
   └─ copyFiles                → package.json、README、LICENSE
4. copyTypesDefinitions → 把类型文件复制到 es/ 和 lib/
5. cleanupTypesDefinitions → 清理临时类型目录
```

### 构建产物结构

```
dist/element-plus/               ← 这就是最终要发布到 npm 的内容
├── package.json                 ← 从 packages/element-plus/package.json 复制
├── README.md
├── LICENSE
├── global.d.ts                  ← Volar 全局组件类型
├── es/                          ← ESM 模块（Tree-shaking 友好）
│   ├── index.mjs                ← 主入口
│   ├── index.d.ts               ← 类型入口
│   └── components/
│       ├── button/
│       │   ├── index.mjs
│       │   ├── index.d.ts
│       │   └── style/
│       │       ├── index.mjs    ← SCSS 样式引入
│       │       └── css.mjs      ← CSS 样式引入
│       └── ...
├── lib/                         ← CJS 模块
│   ├── index.js
│   ├── index.d.ts
│   └── components/...
├── dist/                        ← 完整打包（CDN 场景）
│   ├── index.full.js            ← UMD
│   ├── index.full.min.js
│   ├── index.full.mjs           ← ESM
│   ├── index.full.min.mjs
```

## 三、发布流程

当前的 `scripts/publish.sh` 流程：

```bash
pnpm i --frozen-lockfile    # 安装依赖
pnpm update:version         # 设置版本号（读取 $TAG_VERSION 环境变量）
pnpm build                  # 执行完整构建
cd dist/element-plus        # 进入构建产物目录
pnpm publish                # ★ 发布到 npm（这里就是发布 dist/element-plus/ 这个目录）
```

**核心要点：发布的不是整个仓库，而是 `dist/element-plus/` 这个目录**。该目录里有自己的 `package.json`，是一个完整的 npm 包。

## 四、改造为 `@guwave/ui` 的技术方案

### 需要修改的文件清单

#### 1. 包名和常量（核心）

**`internal/build-constants/src/pkg.ts`** — 所有包名常量的源头：

```typescript
// 改前
export const PKG_PREFIX = '@element-plus'
export const PKG_NAME = 'element-plus'
export const PKG_CAMELCASE_NAME = 'ElementPlus'
export const PKG_CAMELCASE_LOCAL_NAME = 'ElementPlusLocale'
export const PKG_BRAND_NAME = 'Element Plus'

// 改后
export const PKG_PREFIX = '@guwave'
export const PKG_NAME = '@guwave/ui'
export const PKG_CAMELCASE_NAME = 'GuboUI'
export const PKG_CAMELCASE_LOCAL_NAME = 'GuboUILocale'
export const PKG_BRAND_NAME = 'Gubo UI'
```

#### 2. 构建输出目录

**`internal/build-utils/src/paths.ts`** — 输出路径：

```typescript
// 改前
export const epOutput = resolve(buildOutput, 'element-plus')

// 改后（注意 scoped 包名的路径）
export const epOutput = resolve(buildOutput, 'guwave-ui')
```

#### 3. 主包 `package.json`

**`packages/element-plus/package.json`** — 这是最终发布到 npm 的包信息：

```json
{
  "name": "@guwave/ui",
  "version": "1.0.0",
  "description": "Gubo UI Component Library for Vue 3",
  "publishConfig": {
    "access": "restricted",
    "registry": "http://192.168.2.210:7001/"
  }
}
```

#### 4. 构建别名插件

**`internal/build/src/plugins/element-plus-alias.ts`** — 构建时会把 `@element-plus/theme-chalk` 替换为 `element-plus/theme-chalk`，改造后需替换为 `@guwave/ui/theme-chalk`：

这个文件已经通过 `PKG_PREFIX` 和 `PKG_NAME` 常量来动态生成，所以只要改了 `build-constants` 就会自动生效。但要注意 `resolveId.filter` 里的正则也需要同步更新：

```typescript
filter: {
  id: /^@guwave\/theme-chalk/,   // 改为新的 scope 前缀
}
```

#### 5. 发布脚本

**`scripts/publish.sh`** — 修改 cd 路径和发布参数：

```bash
#!/bin/sh
set -e

pnpm i --frozen-lockfile
pnpm update:version
pnpm build

cd dist/guwave-ui
pnpm publish --no-git-checks
cd -

echo "✅ Publish completed"
```

#### 6. CSS 类名前缀（可选，但建议保留 `el-`）

CSS 类名前缀由两处控制：

- **JS 端**：`packages/hooks/use-namespace/index.ts` 中的 `defaultNamespace = 'el'`
- **SCSS 端**：`packages/theme-chalk/src/mixins/config.scss` 中的 `$namespace: 'el'`

**建议保持 `el-` 不变**。原因：

- 改 namespace 需要同时改 JS 和 SCSS 两端，且所有组件的 CSS 类名都会变
- 保持 `el-` 前缀可以兼容 Element Plus 生态的第三方插件和主题
- 如果确实要改（比如改为 `gu-`），需要同步修改以上两个文件

#### 7. workspace 内部包名（按需）

所有 `packages/*/package.json` 中的 `@element-plus/*` 名称只是 workspace 内部引用，不会发布。可以选择改或不改：

- **不改**：构建流程不受影响，省事
- **改**：语义更清晰，但工作量大（涉及所有 import 路径）

## 五、使用方式

改造发布后，用户侧的使用方式：

### .npmrc

```ini
@guwave:registry=http://192.168.2.210:7001/
```

### 安装

```bash
pnpm add @guwave/ui
```

### 使用

```typescript
// 完整引入
import GuboUI from '@guwave/ui'
import '@guwave/ui/dist/index.css'

// 按需引入（配合 unplugin-vue-components 自动导入）
import { ElButton, ElInput } from '@guwave/ui'

// 样式按需引入
import '@guwave/ui/es/components/button/style/css'

app.use(GuboUI)
```

组件名仍然是 `ElButton`、`ElInput` 等（除非你也改了组件前缀），只是安装来源变成了 `@guwave/ui`。

### 配合自动导入（推荐）

如果使用 `unplugin-vue-components`，需要自定义 resolver，因为默认的 `ElementPlusResolver` 会解析到 npm 上的 `element-plus`：

```typescript
// vite.config.ts
import Components from 'unplugin-vue-components/vite'

Components({
  resolvers: [
    // 自定义 resolver
    (componentName) => {
      if (componentName.startsWith('El')) {
        const name = componentName.slice(2) // 去掉 El 前缀
        const kebab = name
          .replace(/([A-Z])/g, '-$1')
          .toLowerCase()
          .slice(1)
        return {
          name: componentName,
          from: '@guwave/ui',
          sideEffects: `@guwave/ui/es/components/${kebab}/style/css`,
        }
      }
    },
  ],
})
```
